# AI Use Statement

AI was used as a generation and design-assistance tool for this coursework project. The workflow treated AI outputs as drafts that required human review, testing, safety checks, and organization.

## How AI Was Used

- Drafted requirements, user stories, architecture notes, and safety policy.
- Generated PlantUML source for use case, class, and activity diagrams.
- Generated Flask, HTML, CSS, and pytest starter files.
- Generated a homepage image prompt and produced a static healthcare intake banner image.
- Added a bounded AI workflow for follow-up question generation and structured symptom summary generation.
- Added safe care-guidance generation that avoids prescriptions and exact medication dosing.
- Refined the final scope by removing optional map search, saved location, and image-recognition/report-image features so the assessed text-based AI workflow stayed clear and testable.
- Added optional profile/history context and health-record structuring so AI use is visible but user-controlled.
- Helped structure CI/CD and deployment evidence.

## Human Review and Control

- The final project structure was selected to match the assessment rubric.
- Generated code was checked for consistency across routes, templates, schema, safety rules, and tests.
- Tests were added for positive behavior, invalid input, and emergency-like safety triggers.
- Tests were added for profile consent, checkup-record structuring, removed optional endpoints, and the core route/API workflow.
- No API keys or private credentials are included in the submission.
- No real patient data should be used in this coursework submission.

## Academic Integrity Boundary

The submitted work is presented as an AI-assisted software development workflow. AI was used to accelerate generation and improve structure, but the final assessment package was organized, verified, and explained by the student.
