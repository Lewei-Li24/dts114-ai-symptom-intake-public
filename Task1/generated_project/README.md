# AI Symptom Intake and Triage Support Website

This generated Flask application demonstrates a non-diagnostic AI symptom intake workflow. Users enter symptoms, answer generated follow-up questions, and receive a structured summary with careful next-step guidance. The app does not diagnose, prescribe, or replace professional medical care.

## Safety Boundary

This website is not a diagnosis tool, does not replace a doctor, and does not provide prescriptions. If symptoms are severe, rapidly worsening, or concerning, users should seek urgent professional medical help.

## Run Locally

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
flask --app app init-db
flask --app app run
```

Open `http://127.0.0.1:5000`.

## AI Tooling

The Task 1 notebook records a file-level generation workflow using APIFree OpenAI-compatible model calls. User-facing AI features require a working APIFree key and show a visible error if the provider is unavailable, rather than silently pretending to be model output. Local safety-rule checks continue to run without the model. The app uses AI for:

- generating follow-up questions from initial symptom input
- producing a structured symptom summary
- producing non-prescriptive care guidance and clinician/pharmacist discussion prompts
- structuring pasted checkup or health-record text into JSON
- optionally using saved profile and recent records during chat intake when the user enables consent
- supporting prompt-designed documentation and live UML generation through `/api/uml/generate`
- generating the homepage healthcare intake banner

Set APIFree variables before running the app:

```bash
set APIFREE_API_KEY=your_key_here
set APIFREE_BASE_URL=https://api.apifree.ai/v1
set APIFREE_MODEL=deepseek-ai/deepseek-v4-flash
```

For local development you may copy `.env.example` to `.env` and fill in your own key. Do not commit `.env`.

If no model key is configured, AI-labelled chat, care guidance, and health-record structuring show a clear configuration error. This makes the coursework evidence honest: live model output is visibly different from local safety rules.

## Care Guidance Boundary

The care guidance feature does not prescribe medication, give exact doses, or recommend antibiotics/prescription drugs. It provides general self-care suggestions, monitoring ideas, visit-preparation checklists, and questions to discuss with a licensed clinician or pharmacist.

## Conversational Intake

The main intake page uses a chat workflow. The assistant asks follow-up questions before generating the summary and care guidance. With `APIFREE_API_KEY` configured, the chat replies can be generated by the model provider; without a key, deterministic fallback questions keep the demo working.

## Personalization

The `/profile` page lets users save local demo profile details, allergies, current medicines, chronic conditions, medical history, and pasted checkup records. The chat page includes a separate consent toggle for whether model-assisted intake may use previous turns and saved profile context.

Map search and image-recognition features were removed to keep the prototype focused on text-based symptom intake, profile management, checkup-text structuring, and non-diagnostic guidance.
