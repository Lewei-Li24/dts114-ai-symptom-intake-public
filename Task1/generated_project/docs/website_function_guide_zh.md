# AI 症状采集与分诊辅助网站：功能与使用指南

## 网站定位

本网站是一个文本式 AI 症状采集与分诊辅助原型。它不做诊断、不提供处方、不替代医生，而是帮助用户在就医前整理症状信息，并生成结构化摘要、非诊断型护理建议和课程要求的 UML 生成证据。

最终版本已移除地图搜索、浏览器定位、图片识别和报告图片生成功能，避免功能发散。当前范围聚焦于：AI 对话式症状采集、健康档案、体检单文字结构化、UML 生成、护理建议、安全提醒、测试与部署证据。

## 核心页面

| 页面 | 路径 | 作用 |
| --- | --- | --- |
| 首页 | `/` | 展示项目目标、流程和 AI 生成 banner 图 |
| 症状采集 | `/symptom-input` | 选择快速采集或已有用户档案，进入 AI 对话 |
| 症状摘要 | `/summary/<session_id>` | 展示结构化症状摘要 |
| 护理建议 | `/care-guidance/<session_id>` | 展示非诊断型护理建议、药物讨论参考和行动计划 |
| 健康档案 | `/profile` | 新建或编辑用户资料，导入体检单文字 |
| UML 生成器 | `/uml-generator` | 输入业务问题，通过 Flask API 调用 APIFree 生成 PlantUML |
| 安全说明 | `/safety-notice` | 展示系统边界、红旗症状和免责声明 |
| About | `/about` | 展示 AI 使用方式和项目说明 |

## 推荐演示流程

1. 打开 `/`，展示首页和自动生成 banner 图。
2. 打开 `/profile`，新建或选择一个健康档案。
3. 在 `/profile/new` 中填写年龄、过敏史、当前用药、慢性病和既往史。
4. 粘贴体检单文字，点击 `Auto-structure Preview`，展示 AI 如何整理为统一 JSON。
5. 打开 `/symptom-input`，选择快速采集或已有用户档案。
6. 在聊天框输入症状，让 AI 逐步追问。
7. 完成采集后查看 `/summary/<session_id>`。
8. 打开 `View Care Guidance`，查看非诊断型护理建议和药物讨论参考表。
9. 打开 `/uml-generator`，输入业务问题并生成 PlantUML，展示 AI-specific tooling。
10. 输入高风险症状，如 `chest pain and trouble breathing`，展示 safety notice。

## AI 能力

- 生成症状追问问题。
- 将对话整理成结构化症状摘要。
- 生成非诊断型护理建议和药物讨论参考。
- 将用户粘贴的体检单文字整理成统一 JSON。
- 在用户授权后，参考已有健康档案和近期记录。
- 根据业务问题生成 PlantUML，用于 SDLC/UML 证据。

## 安全与隐私

- 每页都有免责声明。
- 红旗症状先走本地安全规则，再决定是否调用 AI。
- APIFree key 只在后端环境变量中使用，不暴露给前端。
- AI 输出经过结构化校验和 fallback 处理。
- 用户可以选择是否允许模型参考个人档案和历史记录。
- 本项目只使用匿名演示数据，不应上传真实患者隐私信息。
